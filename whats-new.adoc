== 06 avril 2026



=== Amazon FSX pour NetApp ONTAP



==== Mises à jour d'analyse bien conçues

Workload Factory analyse vos systèmes de fichiers Amazon FSx for ONTAP pour les *volumes NAS inactifs*. L'analyse de configuration identifie les volumes NAS qui ne sont pas activement utilisés et recommande des actions pour optimiser l'utilisation du stockage et réduire les coûts.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Mettre en œuvre des systèmes de fichiers FSx for ONTAP bien architecturés"]



==== Fonctionnalité de table de journal pour les points d'accès S3

La fonctionnalité Journal table permet de gérer le stockage d'objets S3 de FSx ONTAP à grande échelle, facilitant ainsi l'audit des événements d'accès utilisateur et des opérations sur les objets dans vos compartiments. Avec Workload Factory, vous pouvez configurer des analyses automatiques de vos compartiments et tables S3 afin de conserver un historique des événements, affichant tous les objets, leurs détails et les actions des utilisateurs.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["Configurez l'infrastructure de la table de journalisation pour NetApp Workload Factory"]



==== Prise en charge de la gestion des igroups existants

Workload Factory prend en charge la gestion des igroups existants. Vous pouvez ajouter ou supprimer des périphériques de blocs et des initiateurs hôtes afin de garantir la livraison de votre charge de travail déployée dans les délais impartis.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Gérez le stockage bloc dans NetApp Workload Factory"]



==== Prise en charge de la spécification de l'agrégat pour les volumes FlexVol

Workload Factory permet de créer des volumes FlexVol sur un agrégat choisi. Cela vous permet de répartir la charge de capacité et de performance sur l'ensemble des ressources du système de fichiers.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["Créez un FlexVol volume dans NetApp Workload Factory"]



==== Dernier temps d'accès pour les périphériques de blocs

Workload Factory indique le dernier temps d'accès aux périphériques de stockage par blocs. L'heure est basée sur votre dernier accès à l'igroup dans Workload Factory. Savoir quand chaque périphérique de stockage par blocs a été utilisé pour la dernière fois vous permet d'optimiser vos ressources.



=== Charges de travail des bases de données



==== Analyse bien architecturée pour Oracle

L'analyse bien architecturée inclut les nouvelles configurations suivantes pour Oracle :

* *Mises à jour critiques d'Oracle* : évalue les correctifs de sécurité manquants pour les installations de bases de données autogérées Oracle prises en charge.
* *Réplication interrégionale* : vérifie si la réplication interrégionale (CRR) est activée pour vos systèmes de fichiers FSx for ONTAP servant des bases de données Oracle.
* *Instantanés cohérents au niveau des applications* : vérifie si la base de données Oracle est protégée par des instantanés cohérents au niveau des applications avec NetApp SnapCenter.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implémenter des configurations de base de données bien architecturées dans Workload Factory"]



==== Évaluation ponctuelle pour Oracle

Workload Factory propose désormais une évaluation ponctuelle pour analyser la configuration de vos déploiements de bases de données Oracle, à tout moment : nouveaux déploiements, post-migration, dépannage, etc. Vous recevrez des recommandations pour vous aider à optimiser vos configurations de déploiement et à vous conformer aux bonnes pratiques. C'est l'occasion d'utiliser la fonctionnalité bien architecturée sans vous engager pleinement dans le processus de configuration. Vous pouvez évaluer vos bases de données sans stocker d'identifiants dans Workload Factory, sans enregistrer de ressources de base de données ni configurer de connexions externes.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Effectuez une évaluation ponctuelle de vos déploiements de bases de données Oracle"]



==== Amélioration des économies pour plusieurs bases de données Oracle

Nous avons amélioré l'analyse des économies pour Oracle afin de fournir une solution de stockage consolidée pour plusieurs bases de données Oracle sur site sur un seul système de fichiers FSx for ONTAP. Cette consolidation optimise les coûts de stockage en réduisant le nombre de systèmes de fichiers nécessaires pour plusieurs bases de données, ce qui conduit à une amélioration des économies.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Découvrez les économies pour les bases de données détectées sur Oracle sur site"]

=== Charges de travail EDA



==== Recommandations en matière de capacité EDA

Workload Factory analyse désormais automatiquement la capacité SSD de chaque cluster FSx for ONTAP dans votre inventaire EDA et génère des recommandations de capacité lorsque des ajustements sont nécessaires. L'algorithme s'exécute toutes les 24 heures pour chaque cluster.

Lorsqu'une recommandation est identifiée :

* Vous recevez immédiatement un e-mail ou une notification WAD (non incluse dans le résumé hebdomadaire).
* Un indicateur lumineux apparaît sur le cluster concerné dans le tableau des clusters.
* Un résumé sur le tableau de bord Vue d'ensemble affiche le nombre total de clusters avec des recommandations actives.


Sélectionnez un cluster comportant une recommandation active pour consulter la recommandation, qui explique la modification suggérée et la justification.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["Apprenez-en davantage sur les recommandations en matière de capacité"^].



==== Analyse automatisée de la latence de l'IA

L'analyse de la latence inclut désormais l'analyse des agents d'IA.

Pour les scénarios complexes impliquant des données ou une latence de cluster, vous pouvez faire appel à une analyse par agent d'IA pour obtenir :

* Identification détaillée des causes profondes
* Liste des clients EC2 concernés
* Étapes de remédiation spécifiques


L'analyse par agent d'IA nécessite la configuration ARN du modèle Amazon Bedrock et est disponible à la demande pour les scénarios où une analyse de base recommande une investigation plus approfondie. L'agent d'IA peut identifier des problèmes tels que les volumes excessifs de trafic, les configurations de points de montage non optimales, FlexGroup, les besoins de rééquilibrage et les exigences de montée en charge horizontale.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["Apprenez-en davantage sur l'analyse des agents IA"^].

=== Workloads VMware



==== Présentation de la prise en charge de l'assistant de migration pour Google Cloud VMware Engine

NetApp Workload Factory prend désormais en charge Google Cloud VMware Engine. Vous pouvez migrer vos charges de travail VMware sur site vers Google Cloud VMware Engine en utilisant Google Cloud NetApp Volumes comme datastores NFS externes, optimisant les coûts et permettant un meilleur contrôle de votre environnement VMware sans avoir à refactoriser ni à replatformer vos applications.

L'assistant de migration Google Cloud VMware Engine fournit :

* Dimensionnement et conception automatisés des clusters Google Cloud VMware Engine avec prise en charge des types de nœuds VE1 et VE2
* Optimisation de la disposition des machines virtuelles entre les datastores internes vSAN et les datastores NFS externes Google Cloud NetApp Volumes.
* Comparaison détaillée des coûts et analyse du coût total de possession (TCO) entre les configurations utilisant uniquement vSAN et celles optimisées par Google Cloud NetApp Volumes.
* Prise en charge des niveaux de service Standard, Premium et Extreme de Google Cloud NetApp Volumes.
* Déploiement d'infrastructure en tant que code avec Terraform pour un provisionnement automatisé.

=== Amazon FSX pour NetApp ONTAP



==== Mises à jour d'analyse bien conçues

Workload Factory analyse vos systèmes de fichiers FSx pour ONTAP pour les configurations suivantes :

* Capacité SSD sous-utilisée : vérifie si le niveau de capacité SSD est sous-utilisé et recommande de réduire la capacité SSD pour optimiser les coûts.
* Planifiez les sauvegardes de volumes : NetApp Workload Factory améliore la recommandation de configuration et ajoute des options pour activer la sauvegarde FSx for ONTAP ou AWS Backup. FSx for ONTAP backup applique une stratégie par défaut à tous les volumes d'un système de fichiers, créant des sauvegardes quotidiennes conservées jusqu'à 90 jours. AWS Backup vous permet de choisir des périodes de rétention différentes pour chaque volume du plan de sauvegarde.




==== NetApp Workload Factory prend en charge les cas d'utilisation de l'IA et du ML pour la réplication

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["Répliquer les données sur site pour les cas d'utilisation de l'IA et du ML"]

==== Mise à jour de la fonction de gestion automatique de la capacité

Cette mise à jour divise la fonctionnalité en deux fonctionnalités distinctes : gestion automatique de la capacité et notifications de capacité.

La gestion de la capacité ajoute ou supprime automatiquement de la capacité du système de fichiers afin de maintenir des performances optimales. Parmi les nouvelles options figurent les modes incrémental et adaptatif. En mode incrémental, le système augmente la capacité d’un montant fixe à certains seuils ; en mode adaptatif, il utilise des données historiques pour prédire et ajuster la capacité.

Les notifications de capacité vous alertent à chaque modification de capacité et lorsque les systèmes de fichiers atteignent les limites définies, afin que vous puissiez gérer le stockage avant que des problèmes ne surviennent.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/enable-auto-capacity-management.html["Découvrez la gestion automatique de la capacité et les notifications de capacité dans NetApp Workload Factory"]

==== Tableau de bord de synthèse pour la gestion du stockage EDA

En tant qu'administrateur informatique gérant des dizaines de systèmes de fichiers FSx for ONTAP, vous pouvez utiliser le tableau de bord Vue d'ensemble pour répondre aux questions opérationnelles courantes concernant l'état du cluster, son utilisation et l'emplacement des volumes.

L'ancien tableau de bord principal est désormais accessible dans l'onglet *Projet*. Les onglets EDA sont maintenant ordonnés : Vue d'ensemble, Latence, Projet, CI/CD.

==== Analyse automatisée de la latence

L'analyse de latence inclut désormais une analyse de base automatisée. Lorsque des événements de latence sont détectés, le système analyse automatiquement les métriques du centre de délai QoS ONTAP afin d'identifier la source des problèmes de latence.

L'analyse de base évalue les politiques QoS, les performances du disque, les problèmes de réseau et les configurations du cluster afin de fournir des informations immédiates dans la colonne Details du tableau des événements de latence.

=== Amazon FSX pour NetApp ONTAP



==== Prise en charge de la migration des VM de stockage

NetApp Workload Factory prend désormais en charge la migration des machines virtuelles de stockage. Cette fonctionnalité permet la migration des données et des configurations des systèmes de stockage ONTAP depuis des systèmes ONTAP sur site ou des systèmes de fichiers FSx for ONTAP de première génération vers des systèmes de fichiers FSx for ONTAP de deuxième génération. Vous pouvez répliquer les données et les paramètres de configuration des machines virtuelles de stockage vers le nouveau système de fichiers avec une interruption minimale et une perturbation réduite pour les utilisateurs et les applications.

Pour utiliser cette fonctionnalité, link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["créer une relation de réplication"] et sélectionnez *Migration* comme cas d'utilisation. Pour mener à bien le processus de migration, vous devez link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html#cut-over-replication-for-migration-use-cases["basculer la machine virtuelle de stockage et ses volumes répliqués"] immédiatement pour migrer définitivement les données et les paramètres de configuration de la machine virtuelle de stockage vers le système de fichiers FSx for ONTAP cible.
